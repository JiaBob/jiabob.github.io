About my site

This website is my first blog. I have planned to build a website from scratch for a long time, but unfortunatly I did not take this step until I took Course COMP1710 on March 2019. Actually, I have built some website before, which means I do not have to learn website programming fully from this Course. But I still thanks this course for indirectly giving me such opportunity.
The reflection contains three parts: The program view mainly describes the program used for build the website. The content view describes the possible content that the boblog will pubish. The experience view describes what I learn during building this website.

  • Program view

    Apparently the website is static and it is held on github. The frontend only uses Boostrap4 and JQuery. I implement most of components from sracth. However, actually the entire work is not such simple. I programmed Django code for this website as backend. The backend. As it is only on my local computer, I regard this backend as static framework such as Hexo. Hence, there is an admin for content, including article and gallery, adding. If you have used Markdown, you can find this article is exactly Markdown. It is also the work of backend. Besides, I use MySQL for storing those content. To finally make a dynamic website static, I write a python script to crawl my local web server and localize the content so that it can be visited completely as a static HTML files.

    Here are some interesting parts of this work briefly. - CSS3 Animation (href highlight, four direction arrows) - Image Progressive Loading (animation, thumbnail, AWS S3) - Image Browser (gallery, designed from Boostrap4 modal) - Dynamic to Static (python script based on regular expression)

  • Content view

    With respect to the COMP1710 assignment, the website contains a assignment description page(which will be removed after), sitemap.html(which should be .xml), source log, image map, gallery, facebook share button, facebook page feed, etc. In terms of my own blog, the core part is the article. Later on I will update some technical article in terms of my research field and my software development experience.

  • Experience view

    Although, as I said, I have some full-stack website build experience, I still learn a lot this time. I try to use something pure and something that I have never used. The most important one is about image gallery. The pain point is when we put original image on remote server, every time visitor browse the web page they have to wait the image loading. If there are many such kind of image, the loading time will be extremely long. Hence many methods are created along history, such as Spirite image, BASE64 encoding, lazy loading, progressive JPEG, CDN, etc. For my website, I uses a progressive loading strategy, which is frequently used on Medium. For each original image, I put a thumbnail one and a original one on AWS S3. The website could load the thumbnail very quickly and wait for the loading of original one. Then use CSS3 animation to make the loading like a blur-to-vivid process. The second important creation is about the backend. My python crawler script is able to convert a dynamic website into fully static one. With the help of this, the website construction will be extremely convenient. For example, my website contains more than ten pages. But nearly all of them share same navigation bar and footer. If I write the HTML purely by "hand", I have to copy and paste them many many times. Furthermore, if something changed, I have to change all of them one by one. Moreover, there is an admin page on Django backend, which make the article writing and image publish a more friendly process. Because I do not have to do this directly on HTML files.

There are many on going works. For example, making the website more adaptive (e.g. adaptive text), text background animation, multilingual web page.
